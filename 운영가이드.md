# 사무실 도시락 주문판 — 운영 가이드

> 배포·데이터·수정 방법 정리. (2026-07 기준)

## 1. 개요
- **라이브 주소:** https://dosirak.rye502.workers.dev
- **저장소:** https://github.com/mean0000/dosirak_demo
- **호스팅:** Cloudflare Workers (정적 자산 + KV)
- 한 주 단위로 참여자가 도시락 메뉴를 고르고, 관리자가 집계·예산·발주(카톡 복사)를 처리하는 웹앱.

## 2. 구조 (코드 vs 데이터 = 완전 분리)
| 구분 | 위치 | 내용 |
|---|---|---|
| 코드 | GitHub 저장소 | `worker.js`(백엔드), `public/index.html`(화면), `wrangler.toml`(설정) |
| 데이터 | Cloudflare **KV** (`DOSIRAK`) | 인원·메뉴·예산·주문 등 실제 데이터 |

- `worker.js`: `/api/data` 요청은 KV로 처리, 그 외는 `public/`의 정적 파일을 서빙.
- **GitHub엔 코드만, KV엔 데이터만.** 서로 안 섞임.

## 3. 배포 (코드 반영)
- **자동 배포:** `main` 브랜치에 `git push` → Cloudflare가 자동으로 `npx wrangler deploy` 실행 → 반영.
  - 진행 상황: 대시보드 → `dosirak` 워커 → **Deployments / Builds** 탭.
- **수동 배포(선택):** 프로젝트 폴더에서 로그인 후 `npx wrangler deploy`.
- 빌드 설정: Build command = (비움), Deploy command = `npx wrangler deploy`, Root = `/`.

## 4. 데이터 저장 (Cloudflare KV)
KV는 Cloudflare가 관리하는 **key-value 데이터베이스**(서버리스). 데이터는 Cloudflare 글로벌 네트워크에 저장됨 (내 PC·GitHub 아님).

- **네임스페이스:** `DOSIRAK` (id: `db67e57a4d80495f84a074ff539d8cac`)
- **key 구조:**
  | key | 값 |
  |---|---|
  | `config` | 전체 설정 (인원·메뉴·예산·머리말 등) |
  | `wk:<주월요일>:days` | 그 주 주문일 (예: `wk:2026-07-06:days`) |
  | `wk:<주월요일>:p:<인원id>` | 그 사람의 그 주 선택 |
- **직접 보기/편집:** 대시보드 → **Storage & Databases → KV → `DOSIRAK`**.
- 무료 한도(하루 읽기 10만·쓰기 1천)로 이 용도엔 충분.

## 5. 데이터 안전 ⚠️
- ✅ **코드를 아무리 수정·재배포해도 KV 데이터는 안 날아감.** (코드만 교체됨)
- 데이터가 사라지거나 어긋나는 경우는 아래뿐:
  1. KV 네임스페이스를 **삭제**할 때
  2. `wrangler.toml`의 KV **`id`를 다른 값으로 변경**할 때 (기존 데이터는 남지만 앱에서 안 보임)
  3. 대시보드에서 **key를 직접 삭제**할 때
  4. 코드에서 **데이터 구조(key 이름·config 스키마)를 크게 변경**할 때 → 삭제는 아니고 호환성 문제 (이런 변경 전엔 마이그레이션 확인 필요)

## 6. 자주 하는 작업
- **메인 화면 주 변경:** 관리자 → 주간 현황에서 ‹ › 로 원하는 주로 이동 → **「📌 이 주를 메인 화면에 저장」**. 참여자 메인 화면이 그 주로 고정됨. 「자동으로 되돌리기」를 누르면 달력 기준 이번 주가 자동 표시(월요일마다 자동 전환)로 복귀. (설정은 `config.activeWeek`에 저장)
- **기능/디자인 수정:** `public/index.html` 등 편집 → `git commit` → `git push` → 자동 배포.
- **로컬 미리보기:** `public/index.html`을 브라우저로 열면 됨 (KV 없이 데모 모드로 동작).
- **주소 유지:** 현재 `dosirak.rye502.workers.dev` 사용. 커스텀 도메인은 회사 도메인이 Cloudflare에 있을 때만.

## 7. 저장 모드 표시
- **☁️ 클라우드 저장:** KV 연결됨 — 어느 기기서든 같은 데이터.
- **💾 데모(로컬):** KV 미연결/오프라인 — 브라우저 `localStorage`에만 저장.
